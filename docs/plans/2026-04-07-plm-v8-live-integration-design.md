# PLM v8 真实集成与执行深化设计

## 目标

在现有 `PLM v7` 企业级变更管理模块基础上，继续推进到更接近真实企业现场的 `PLM v8`：

- 把外部系统边界从“建模与事件记录”升级到“真实连接器框架”
- 把实施协同从“轻量任务板”升级到“模板化执行工作区”
- 把 BOM / 文档 / 图纸能力从“对象视图”升级到“可比较、可验证、可发布”

本阶段仍然不追求一次性做完整 ERP/MES/PDM/CAD 双向生产集成，而是交付一套：

- 可以稳定扩展的连接器框架
- 可以执行的实施协同工作区
- 可以支撑后续集成的对象发布语义

## 范围

### 一、外部连接器框架

交付：

- `plm_connector_registry`
- `plm_connector_job`
- `plm_connector_dispatch_log`
- `plm_external_ack`
- 统一连接器接口（ERP / MES / PDM / CAD）
- 出站任务创建、状态跟踪、失败重试、幂等键

目标：

- 从当前“单据生命周期 -> 集成记录/事件”升级为“单据生命周期 -> 连接器任务 -> 外部回执”

### 二、实施协同工作区

交付：

- 任务模板
- 任务依赖
- 任务阻塞原因
- 验证证据（文本 / 附件）
- 关闭前验收清单

目标：

- 让“实施、验证、关闭”不再只是状态流转，而是有过程资产

### 三、对象深化 v2

交付：

- BOM 差异层级视图
- 文档/图纸发布状态
- 文档 revision 历史
- 变更对象的受控发布语义

目标：

- 支持“影响哪些对象、当前发布到哪里、是否完成发布”

### 四、驾驶舱 v2

交付：

- 连接器任务健康度
- 外部系统失败分布
- 实施执行率
- 验收完成率
- 关闭阻塞来源分析

目标：

- 管理层能看懂为什么变更没有真正落地到外部系统

## 架构

### 后端

新增核心服务：

- `PlmConnectorOrchestrationService`
- `PlmConnectorDispatcher`
- `PlmConnectorAckService`
- `PlmImplementationTemplateService`
- `PlmImplementationEvidenceService`
- `PlmPublicationService`

新增接口方向：

- 连接器任务查询/重试
- 外部回执上报
- 实施模板与依赖配置
- 任务证据上传/查询
- 发布状态与 revision 历史查询

### 前端

详情工作区新增区块：

- 连接器任务与回执
- 实施任务模板与依赖图
- 任务证据与验收
- BOM 差异层级视图
- 文档发布状态 / revision 历史

工作台 / 驾驶舱新增：

- 外部系统失败热点
- 待回执连接器任务
- 实施阻塞排行

## 并行拆分

### Lane A：后端连接器框架

- 连接器 registry / job / ack 表
- Dispatcher / retry / idempotency
- ERP / MES / PDM / CAD 连接器 stub

### Lane B：后端实施协同深化

- 模板
- 依赖
- 证据
- 验收清单

### Lane C：前端工作区深化

- 集成任务面板
- 实施依赖与证据面板
- BOM diff / document publication 面板
- 驾驶舱 v2

## 完成标准

1. 单据提交后能生成真实连接器任务，而不只是记录边界事件
2. 失败连接器任务可重试、可查看最近回执
3. 实施/验证/关闭有模板、证据、验收清单
4. 详情页能明确显示对象发布状态和外部推进状态
5. 驾驶舱能回答：
   - 哪个系统失败最多
   - 哪些单据卡在外部同步
   - 哪些实施任务阻塞关闭
