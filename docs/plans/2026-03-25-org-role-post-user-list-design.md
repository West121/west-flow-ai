# 部门角色岗位关联用户查看设计

## 目标

在部门、角色、岗位列表页的每一行操作中新增“关联用户”入口，点击后就地查看该实体关联的用户列表，不跳转离开当前列表页。

## 方案

- 前端在部门、角色、岗位列表页复用一套只读 Dialog。
- 后端分别为部门、角色、岗位提供轻量查询接口，返回关联用户数组。
- 用户列表字段统一为：姓名、账号、部门、岗位、状态。
- 权限沿用系统用户可见范围，避免通过关联查询绕过数据权限。

## 取舍

- 不跳转到用户列表页，避免打断当前操作流。
- 先不做分页和搜索，当前场景以快速查看为主，后续如用户量过大再升级。
- 不改详情页入口，避免引入和本需求无关的页面结构调整。
