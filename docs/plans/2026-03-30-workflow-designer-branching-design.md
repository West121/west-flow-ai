# 流程设计器分支与连线交互设计

## 目标

把流程设计器里的“条件分支”从节点面板主导改成连线主导，降低条件配置路径长度，并清理当前画布上误导性的加号入口。

## 当前问题

### 1. 连线上加号过度暴露

当前所有连线统一使用 `quickInsert` 边组件，导致：

- 条件网关出边中段出现“新增节点”加号
- 分支汇聚中间连线也出现加号
- 用户无法区分“这条线是结构连接”还是“这条线承载条件”

这会直接造成画布噪音，并误导用户认为中间汇聚点也能插入审批结构。

### 2. 条件配置路径过长

当前条件配置主要集中在条件节点右侧面板：

- 用户先选中网关
- 再在面板里找分支列表
- 再在列表中修改某条连线的名称/条件/优先级

这和实际心智不一致。条件实际挂在连线上，入口却在节点里。

### 3. 新增条件分支路径不自然

当前条件网关下方的加号只能“在线上插入节点”，不能直接“新增一条条件分支”。用户如果想增加一个分支，需要先创建结构，再回到节点面板重新配置条件，成本偏高。

## 设计原则

### 1. 条件属于连线，不属于网关节点

- 排他/包容网关只保留网关级策略
- 每条出边独立承载：
  - 分支名称
  - 条件表达式
  - 条件类型
  - 包容分支优先级

### 2. 网关只负责“路由规则”，不负责“逐条分支编辑”

- 排他网关保留：
  - 默认分支
- 包容网关保留：
  - 汇聚策略
  - 默认分支
  - 必选分支数

### 3. 画布操作要贴近用户意图

- 点击分支连线，直接编辑该分支条件
- 选中条件网关，直接新增条件分支
- 非结构性关键连线不显示“在线插节点”按钮

## 交互方案

## 1. 连线支持选中与属性编辑

设计器增加 `selectedEdgeId`，允许画布进入“连线编辑态”。

交互规则：

- 点击节点 -> 进入节点配置面板
- 点击连线 -> 进入连线配置面板
- 点击空白区域 -> 清空节点和连线选中

当选中的是排他/包容网关的出边时，右侧属性区切换成“分支配置”。

## 2. 分支配置迁移到连线面板

新增“连线配置”面板，包含：

- 分支名称
- 条件类型
  - 字段比较
  - 安全公式
  - 手写表达式
- 条件字段 / 运算符 / 值
- 包容分支优先级

默认分支不显示条件编辑，只提示：

- 这是默认分支
- 未命中其他分支时自动进入

## 3. 条件网关底部增加“新增条件分支”

当选中节点是：

- `condition`
- `inclusive` 且 `gatewayDirection === 'SPLIT'`

节点底部出现主操作按钮：`新增条件分支`

点击后自动：

1. 从当前网关右下方创建一个新的审批节点
2. 新增一条从网关指向该节点的出边
3. 自动为新分支生成默认名称
4. 自动选中新出边并切换到连线配置面板

## 4. 加号显示规则重构

不再让所有边统一显示 `quickInsert`。

只允许以下连线显示插入节点入口：

- 普通主干连线
- 非条件网关出边
- 非汇聚目标前的回流连线

以下连线隐藏加号：

- 排他网关出边
- 包容网关分支出边
- 汇聚前的横向/回流线
- 指向 `JOIN` 网关的入边

## 状态模型调整

### 新增状态

- `selectedEdgeId: string | null`

### 新增动作

- `setSelectedEdgeId`
- `updateEdgeDraft`
- `addBranchOnGateway`

### 当前状态保持不变

- `WorkflowEdge.data.condition`
- `WorkflowEdge.data.priority`
- `WorkflowConditionNodeConfig.defaultEdgeId`
- `WorkflowInclusiveNodeConfig.defaultBranchId`

这次主要调整的是交互与状态入口，不重写 DSL。

## 实现边界

### 本次实现

- 选中连线并编辑分支条件
- 条件网关新增条件分支
- 隐藏不该出现的加号
- 节点面板只保留网关级设置

### 暂不实现

- 在连线附近做浮层原地编辑
- 分支拖拽重排
- 连线条件批量复制
- 自动重排整个分支布局

## 风险与处理

### 1. 现有表单 schema 与测试依赖节点面板分支数组

处理：

- 保持 `condition.branches` / `inclusive.branches` 仍由表单生成
- 但数据来源改成“当前选中网关的出边集合”
- 原有 DSL 结构不变

### 2. React Flow 默认没有独立“边选中面板”模式

处理：

- 在设计器 store 中显式维护 `selectedEdgeId`
- 由 `pages.tsx` 统一决定属性面板是节点模式还是连线模式

### 3. 分支新增后画布可能拥挤

处理：

- 第一版只做稳定的默认偏移布局
- 不做自动整图重排

## 成功标准

- 条件网关出边的条件可直接在连线面板编辑
- 条件网关中间与汇聚位置不再出现误导性加号
- 条件节点底部可直接新增条件分支
- 排他/包容网关节点面板显著简化，只保留网关级设置
- 现有 DSL 测试和设计器测试能继续通过
