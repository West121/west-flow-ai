# 审批平台能力收口设计

## 背景

当前平台已经具备较强的审批运行态能力：

- 认领、同意、驳回、退回、转办、委派、加签、减签、撤销、催办、拿回、离职转办
- 会签模式：串签、并签、或签、票签
- 协同事件、抄送、时间旅行、终止策略
- 审批意见配置、自动提醒、超时自动处理

但从“中国式审批平台”的完整能力看，仍存在以下空缺：

1. 缺少日常待办的批量审批与批量处理
2. 缺少督办、会办、阅办、传阅等正式审批协同模式
3. 缺少电子签章/签字能力
4. 缺少更完整的退回语义
5. 缺少完整的 SLA 升级链路

这些能力横跨：

- 运行态任务引擎
- 流程 DSL 与设计器
- 工作台前端
- 审批详情与轨迹
- 通知/协同与审计

如果不做分层拆分，极易互相冲突。

## 目标

在不推翻现有运行态主链的前提下，将审批平台增强到更接近国内企业常见平台的程度，完成以下能力闭环：

1. 支持批量审批、批量已读、批量认领
2. 支持督办、会办、阅办、传阅四类协同审批模式
3. 支持电子签章第一版闭环
4. 支持发起人、上一步、任意已走人工节点三类退回目标
5. 支持 SLA 的时限、自动提醒、升级、审计

## 设计原则

### 1. 先补运行态语义，再补界面

所有能力先在后端建立明确语义：

- 动作类型
- 事件类型
- 可用性判断
- 审计记录

前端只消费后端能力，不自己发明语义。

### 2. 不破坏现有单任务主链

批量审批、会办、督办等能力必须作为增量能力接入，不得破坏：

- 当前单任务审批动作
- 现有审批详情时间轴
- 已经上线的协同、抄送、加签、离职转办

### 3. 设计器配置必须和运行态一致

新增能力只允许通过流程 DSL 和工作流管理配置生效，不允许前端工作台出现“运行时特供能力”。

### 4. 签章先做第一版可用闭环

第一版电子签章仅支持：

- 审批节点签章
- 签章意见与附件绑定
- 审批详情回显
- 审计留痕

暂不做复杂合同场景的多位置盖章、骑缝章和 PDF 版式落章。

## 功能分层

## 一、运行态能力层

### 1. 批量审批

第一版支持：

- 批量认领
- 批量已读
- 批量同意
- 批量驳回

约束：

- 仅支持同一动作批处理
- 批量同意/驳回使用统一意见
- 仅处理当前筛选结果中显式勾选任务
- 不做跨页全选

运行态接口新增：

- 批量动作可用性
- 批量动作执行结果
- 单项失败明细

### 2. 更完整的退回语义

当前已有：

- 驳回到上一步
- 驳回到指定节点
- 退回到上一步人工节点

本次统一成三类目标：

- `INITIATOR`：发起人
- `PREVIOUS_USER_TASK`：上一步人工节点
- `ANY_HISTORIC_USER_TASK`：任意已走人工节点

并明确区分：

- `驳回`
- `退回`

两者在轨迹、意见和重审策略上的语义。

### 3. SLA 升级链路

当前已有：

- `timeoutPolicy`
- `reminderPolicy`
- 自动提醒
- 超时自动处理

本次补齐：

- 升级规则 `escalationPolicy`
- 升级目标：用户 / 角色 / 部门负责人
- 升级动作：提醒、转派、抄送督办、升级审批人
- 升级审计事件

第一版不做工作日历与节假日计算。

## 二、协同审批模式层

### 1. 督办

定义：

- 督办人不直接替代审批人
- 督办人负责关注超时、推进和催办
- 督办事件进入审批轨迹

### 2. 会办

定义：

- 会办人给出正式会办意见
- 会办结果参与主审批节点判断
- 与普通协同批注区分

### 3. 阅办

定义：

- 必须签收或已阅
- 可要求阅办意见
- 不直接作为审批结论

### 4. 传阅

定义：

- 形成顺序或并行的阅读链
- 支持阅读状态回显
- 作为正式流程节点类型，而不是普通抄送

## 三、签章层

### 1. 节点签章配置

审批节点支持配置：

- 是否要求签章
- 签章类型
- 签章材料来源

### 2. 运行态签章结果

运行态补充：

- 签章状态
- 签章人
- 签章时间
- 签章摘要/图片引用

### 3. 审批详情回显

审批详情和时间轴展示：

- 该节点是否已签章
- 签章意见与附件

## 四、设计器与工作台收口

### 1. 设计器

设计器新增配置入口：

- 节点批量处理兼容标记
- 退回目标策略
- SLA 升级策略
- 督办/会办/阅办/传阅节点
- 审批节点签章要求

### 2. 工作台

工作台新增：

- 多选列与批量动作条
- 督办/会办/阅办/传阅动作入口
- SLA 状态展示
- 签章动作入口与回显

### 3. 审批详情

审批详情统一展示：

- 督办/会办/阅办/传阅事件
- SLA 触发事件
- 退回语义
- 签章结果

## 分阶段路线

### Phase 1

- 批量审批
- 完整退回语义
- SLA 事件与升级模型

### Phase 2

- 督办
- 会办
- 阅办
- 传阅

### Phase 3

- 电子签章闭环

### Phase 4

- 设计器、工作台、审批详情统一收口
- 最终回归

## 验收标准

1. 工作台可以对勾选任务执行批量认领/已读/同意/驳回
2. 流程节点支持配置并在运行态生效：
   - 退回策略
   - SLA 升级
   - 督办/会办/阅办/传阅
   - 签章要求
3. 审批详情时间轴能看到：
   - 批量动作结果
   - 协同审批模式事件
   - SLA 触发和升级记录
   - 签章记录
4. 全链路测试通过：
   - 后端运行态专项
   - 前端工作台/设计器专项
   - 至少一条真实审批链路验收
