# 微信小程序平台架构设计

## 目标

为现有 `west-flow-ai` 平台新增独立微信小程序端，覆盖与移动端同范围的一期能力：

- 登录与会话态
- 工作台待办 / 已办 / 我发起
- 审批详情与审批动作
- AI Copilot
  - 文字问答
  - 图片上传
  - 语音输入
- 流程图回顾

本次设计的核心约束：

- 不改造现有 `frontend/` Web 应用为跨端单 UI。
- 不直接复用 `apps/mobile/` 的 React Native 渲染层。
- 流程图回顾在微信小程序中使用 `web-view` 承载 H5 只读播放器。
- 小程序端沿用现有后端 API、鉴权与业务模型。

## 技术选型

### 小程序框架

- `Taro`
- `React`
- `TypeScript`

选择原因：

- 当前平台已有 `frontend/` 与 `apps/mobile/` 的 React / TypeScript 体系，新增小程序端继续沿用 React 心智成本最低。
- `Taro` 对微信小程序的页面、路由、上传、录音、`web-view` 能力支持成熟。
- 相比原生微信小程序和 `uni-app`，更容易与现有 monorepo、共享类型与业务逻辑对齐。

### UI 方案

- Taro 原生组件
- 自定义业务组件层

不引入重型整套小程序 UI 库，避免视觉语言和现有平台割裂。小程序端单独适配微信交互习惯，但保持同样的产品信息结构。

### 数据与状态

- `TanStack Query`
- `Zustand`
- `React Hook Form`
- `Zod`

### 流程图回顾

- 小程序原生 `web-view`
- 承载 H5 只读播放器页面

原因：

- 你已经明确接受 `web-view / H5` 方案。
- 现有流程图回顾具备成熟的只读图、时间轴和动画逻辑，H5 承载最稳。
- 小程序端只负责导航、参数传递和容器页面，不重写图引擎。

## 仓库结构

- `apps/weapp/`

- `packages/shared-types/`
- `packages/shared-workflow/`
- `backend/`

- `frontend/`
- `apps/mobile/`
- `services/`

## 共享边界

### 可以共享

- 后端 REST API
- DTO / 类型定义
- 流程图回顾的纯数据逻辑
- AI Copilot 消息、附件、工具卡片的数据结构
- 审批详情与工作台的纯数据格式化逻辑

### 不共享

- Web DOM 组件
- React Native 组件
- 小程序页面布局
- 小程序端流程图渲染层

## 功能拆解

### 登录与会话

- 使用现有 `/api/v1/auth/login`
- 登录后缓存 token、当前用户信息
- 小程序端请求拦截器统一补齐 Bearer Token

### 工作台

- 待办 / 已办 / 我发起三段
- 首页统计摘要
- 任务列表进入审批详情

### 审批详情

- 表单回显
- 时间轴摘要
- 审批动作
- 打开流程回顾页

### 流程图回顾

- 小程序页面只做容器
- `web-view` 打开专门的 H5 流程回顾页面
- 页面参数包含：
  - 任务 / 业务单标识
  - 当前登录 token 或短期访问票据
  - 回顾默认播放位置

### AI Copilot

- 文字问答
- 图片上传
- 录音输入 / 语音上传
- OCR / ASR 继续复用现有后端能力

## 流程图回顾集成方案

### H5 承载页面

新增一个专门给移动端 / 小程序查看的 H5 流程回顾路由：

- 只保留只读播放器
- 不承载 Web 后台壳层
- 聚焦：
  - 流程图
  - 时间轴
  - 播放控制

### 鉴权方式

小程序 `web-view` 不直接信任主页面内存态，需要单独设计：

- 优先方案：进入回顾前由后端换取短期 ticket，再把 ticket 拼进 H5 URL
- H5 用 ticket 拉取详情和流程回顾数据

这样避免：

- 直接把长期 Bearer Token 暴露在 URL 中
- 小程序和 H5 的 cookie / token 同步复杂化

## AI 能力边界

### 首期支持

- 文本问答
- 图片上传识别
- 语音录音 / 上传转写

### 首期不做

- PDF 复杂预览
- 多附件复杂管理
- 小程序内富文本对话编辑器

## 风险点

### 1. `web-view` 与主小程序鉴权链

必须通过短期访问票据或受控跳转方案解决，不能直接在 URL 上长期暴露主 token。

### 2. 微信录音与文件上传差异

小程序端录音、上传、文件预览接口与 Web / RN 不同，需要单独适配。

### 3. 视觉风格一致性

小程序端不能照搬 Web 或 RN 的器件，必须按微信端的可用性和性能约束重组。

## 验收标准

- `apps/weapp` 可以独立安装、编译和启动微信开发者工具预览
- 登录、工作台、审批详情、审批动作链路可用
- AI Copilot 支持文字、图片、语音
- 流程图回顾可通过 `web-view` 进入并正常播放
- 小程序端只读业务主链可完整闭环
