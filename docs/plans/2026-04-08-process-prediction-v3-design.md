# 流程预测 v3 设计

**目标**

把现有流程预测从“详情页的辅助信息”升级成“企业级审批运营能力”，覆盖：
- 更细的预测精度分层
- 更强的 AI 解释与建议
- 面向首页的预测运营驾驶舱
- 基于预测结果的自动动作
- 为未来模型化预测保留稳定数据基础

## 当前基础

现有 `v1 / v2` 已具备：
- 预计完成时间、剩余时长、超期风险、置信度
- 下一节点候选、预测依据、建议动作
- 工作台风险视图
- 流程图预测路径高亮
- 详情页 AI 增强解释（高风险且样本足够时）

但仍然存在三个核心短板：
1. 样本分层还不够细，当前命中精度仍以流程/节点为主。
2. 预测信息分散在详情和列表，缺少“运营面板”和趋势视角。
3. 预测结果还停留在“展示层”，没有稳定触发提醒、催办、协同动作。

## 设计原则

1. **规则/统计主预测不变**
- 数值结果继续由确定性逻辑计算。
- AI 不参与时长、概率、风险等级主值推导。

2. **AI 只做解释层与建议层**
- AI 负责把结构化预测结果转成更自然的说明。
- AI 负责输出催办建议、瓶颈归因、流程优化建议。
- AI 失败时必须稳定回退到规则化解释。

3. **运营优先于炫技**
- 优先让首页、工作台、图谱和自动动作真正能用。
- 模型化预测作为后续增强，不阻塞当前可用性。

4. **样本治理先行**
- 精度改进先依赖更细的样本分层与异常清洗。
- 没有稳定样本治理，就不做激进模型化。

## 方案比较

### 方案 A：继续扩规则与统计（推荐）
- 深化样本分层：流程、节点、办理人、组织、业务类型、工作日/节假日
- 增加分位数校准、异常值清洗、趋势汇总
- AI 只增强解释

优点：
- 稳定、快、可解释
- 与现有实现连续性最好
- 易于审计和回归测试

缺点：
- 对极复杂分支和稀疏样本的适应性有限

### 方案 B：直接上模型化预测
- 用历史任务与实例数据训练统计/ML 模型
- 由模型给出剩余时长和超期概率

优点：
- 理论上更灵活
- 长期潜力更大

缺点：
- 当前历史样本质量、分布和规模未验证
- 调优、回归、解释、治理成本高
- 不适合作为当前主线

### 方案 C：AI 直接做全预测
- 把流程结构、实例轨迹、历史样本摘要直接交给大模型

优点：
- 实现表面上最快

缺点：
- 不稳定、不可审计、成本高、延迟高
- 不适合企业级主预测

**结论：采用方案 A，保留方案 B 作为未来演进方向。**

## v3 范围

### 1. 预测精度 v3

新增样本分层维度：
- 流程定义 key
- 当前节点 id
- 办理人 userId
- 办理组织（部门/岗位）
- 业务类型
- 工作日 / 非工作日 / 节假日前后

新增治理逻辑：
- 中位数 + p75 + p90
- 异常值清洗（极端超长、极端超短）
- 样本衰减（越近的样本权重更高）
- 样本不足时的回退链路明确化

新增输出：
- `currentNodeDurationP90Minutes`
- `sampleTier`
- `outlierFilteredSampleSize`
- `workingDayProfile`
- `organizationProfile`

### 2. AI 解释 v3

把现有增强说明升级为 3 类输出：
- `narrativeExplanation`
- `bottleneckAttribution`
- `optimizationSuggestions`

AI 输入继续使用结构化上下文：
- 当前预测值
- 历史样本画像
- 当前节点 / 候选节点
- 风险级别
- 高风险原因
- 任务负载与超期信号

AI 输出要求：
- 不改写数值
- 不发明节点或人员
- 失败时回退到规则化文案

### 3. 预测运营驾驶舱

在首页新增运营面板：
- 今日高风险待办
- 今日预计超期
- 风险分布
- 超期趋势（近 7/14 天）
- 节点瓶颈排行
- 高风险流程 Top N

支持筛选维度：
- 流程类型
- 部门/组织
- 风险等级
- 时间窗口

### 4. 自动动作

基于预测结果触发：
- 高风险自动催办
- SLA 临近自动提醒
- 下一审批人预提醒
- 高风险协同建议动作（仅生成建议，不自动执行写操作）

自动动作分级：
- `ADVISE_ONLY`
- `NOTIFY`
- `AUTO_URGE`

第一阶段只开放：
- 通知和催办
- 不自动执行审批/转办/退回

### 5. 模型化预测预备层

当前不做主模型上线，但要为后续模型化保留：
- 样本快照表 / 聚合表
- 训练特征字段稳定化
- 预测结果审计字段
- 离线评估入口

这样未来若样本成熟，可以增加：
- 统计模型
- 回归模型
- 生存分析/剩余时间模型

## 架构设计

### 后端

新增/扩展模块：
- `RuntimeProcessPredictionService`
  - 继续负责主预测
  - 增加样本分层、异常清洗、分位数治理
- `RuntimeProcessPredictionAnalyticsService`
  - 汇总趋势、风险分布、瓶颈排行、Top N
- `RuntimeProcessPredictionAutomationService`
  - 负责基于预测的通知/催办策略
- `RuntimeProcessPredictionAiNarrationService`
  - 升级成 AI 解释 v3

### 前端

新增/扩展：
- 工作台首页运营驾驶舱模块
- 列表风险视角加强
- 详情页预测卡升级
- 图谱与时间轴路径级高亮增强

## 数据流

1. 运行时详情 / 列表请求命中预测服务
2. 预测服务返回结构化主预测结果
3. 详情页按条件调用 AI 增强解释
4. 工作台聚合接口生成运营面板数据
5. 自动动作服务读取高风险结果并写通知/催办记录

## 错误处理

- 样本不足：明确显示回退层级，不假装高置信度
- AI 解释失败：回退规则说明，不影响详情主接口成功
- 自动动作失败：记录审计事件，不影响主流程
- 驾驶舱聚合失败：单卡降级，不拖垮首页

## 测试策略

- 分位数计算

- 风险筛选/排序

- AI 解释 v3
- 路径级图谱增强

- 模型化预测预备数据层

## 成功标准

- 详情页预测结果比 v2 更稳定、解释更清楚
- 首页可以直接识别高风险待办和瓶颈节点
- 高风险任务能自动形成通知/催办动作
- 未来可无缝接入统计/ML 模型，而不推翻当前架构
