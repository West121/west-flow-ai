# PLM 企业级能力路线图

## 当前结论

当前系统已经达到：

- 企业级可用的 `PLM 变更管理模块`
- 支持 `ECR / ECO / 物料主数据变更`
- 支持结构化受影响对象、版本差异、实施任务、验证关闭、统计分析、AI 查询

但还没有达到：

- 完整企业级 `PLM 平台`

当前最关键的差距，不再是“有没有审批闭环”，而是以下六层业务深度。

## 一、BOM 深度对象模型

### 已有

- 统一对象主数据
- 对象版本
- 单据对象链接
- 受影响对象清单

### 缺失

- 真正的 BOM 头/行结构
- 多层 BOM 展开
- 替代料 / 可选件 / 生效条件
- BOM 差异对比
- 变更影响传播
- 变更前后结构快照

### 完成标准

- 可查看单据影响的 BOM 层级结构
- 可比较变更前后 BOM 差异
- 可识别新增、删除、替换、属性变更
- AI 能回答“这个变更影响哪些层级、哪些物料”

## 二、图纸 / 文档 / PDM 深度对象

### 已有

- `DOCUMENT / DRAWING` 类型对象
- 基础版本记录

### 缺失

- 图纸与文档的专属元数据模型
- 文件版本关系
- 文件签入/签出语义
- 审批后发布版 / 受控版
- 图纸、说明书、工艺文件关联
- 文档预览与附件工作区

### 完成标准

- 文档/图纸对象可独立查看历史版本
- 单据可显示受影响图纸与文档清单
- 可区分工作版、受控版、作废版
- AI 能回答“这张 ECO 影响哪些图纸/文件”

## 三、版本与配置管理

### 已有

- revision diff
- before / after version 基础字段

### 缺失

- 正式基线
- 配置快照
- 版本发布批次
- 多对象统一版本集
- 版本比较视图
- 变更生效配置

### 完成标准

- 支持基线创建和查看
- 支持按基线查看对象版本集
- 支持单据与基线联动
- 支持 AI 解释“当前差异相对哪个基线”

## 四、实施任务编排深化

### 已有

- 轻量实施任务
- 状态流转
- 验证/关闭前置校验

### 缺失

- 任务模板
- 串并行编排
- 依赖关系
- 工艺/制造/质量专属任务类型
- 验证记录与附件
- 关闭前任务验收清单

### 完成标准

- 可配置实施任务模板
- 支持依赖和阻塞原因
- 支持验证证据、结论、附件
- 关闭时有明确验收证据链

## 五、权限与组织模型

### 已有

- 通用角色/权限体系
- 单据 owner 维度权限

### 缺失

- PLM 专属角色
- 对象级权限
- 产品线/事业部/域权限
- 变更经理、工艺、质量、制造、配置管理员职责边界
- 单据、对象、基线、任务的分层授权

### 完成标准

- 定义并落地 PLM 角色矩阵
- 支持对象级/域级访问控制
- 报表与大盘支持管理视角和业务视角区分

## 六、管理驾驶舱与分析

### 已有

- dashboard analytics
- 分布、趋势、任务告警、owner ranking

### 缺失

- SLA / 周期分析
- 瓶颈节点分析
- 组织/产品线视角
- 关闭率 / 超期率 / 返工率
- 实施效率
- 导出与管理看板

### 完成标准

- 管理层可以看到变更效率和风险热区
- 业务层可以看到待处理瓶颈和超期分布
- AI 能回答“哪个部门积压最多、哪个环节最慢”

## 分阶段路线

### v5：对象与权限基础深化

目标：

- BOM 头/行结构
- 文档/图纸深度元数据
- PLM 专属角色矩阵
- 对象级/域级授权

完成后定位：

- 从“变更管理模块”升级为“带对象约束的 PLM 工作台”

### v6：版本与实施深化

目标：

- 基线 / 配置快照
- BOM 差异比较
- 实施任务模板与依赖
- 验证证据链

完成后定位：

- 从“工作台”升级为“可追踪的变更执行系统”

### v7：经营分析与外部系统耦合

目标：

- 管理驾驶舱
- 组织/产品线/SLA 维度分析
- 附件预览与受控发布
- 为未来 PDM / ERP / CAD 集成预留边界

完成后定位：

- 接近完整企业级 PLM 平台骨架

## 推荐执行顺序

1. 先做 `v5`
   - 因为对象模型和权限模型是后续所有能力的地基
2. 再做 `v6`
   - 因为基线、差异、实施深化依赖对象结构
3. 最后做 `v7`
   - 因为经营分析要建立在稳定业务模型之上

## 当前建议

下一批实现，优先做：

1. `plm_bom_master / plm_bom_line`
2. `plm_document_master`
3. `plm_object_acl / plm_domain_acl`
4. PLM 专属角色矩阵
5. 前端对象工作区与 BOM 展开视图
