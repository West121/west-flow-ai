# 审批平台剩余能力与缺口 Backlog

## P0：必须补齐

### 1. 主子流程前端 E2E

- 覆盖父流程等待子流程、子流程完成后父流程恢复
- 覆盖父子审批详情、流程图、时间轴、结构链接一致性
- 覆盖子流程 + 终止 / 唤醒 / 撤销等关键状态联动

### 2. 复杂审批动作前端 E2E

- 督办 / 会办 / 阅办 / 传阅
- 会签下加签 / 减签 / 驳回 / 退回
- 批量认领 / 批量同意 / 批量驳回 / 批量已读
- 复杂动作后的列表、详情、时间轴、流程图一致性

### 3. 第一轮正式压测

- 读场景：待办列表、审批详情、时间轴、流程图回顾
- 写场景：认领、同意、驳回、退回、加签、减签、转办、委派
- 批量场景：批量认领、批量同意、批量驳回
- 调度场景：SLA 提醒、SLA 升级、编排器处理

## P1：上线前建议补齐

### 4. 外部送达通道联调

- 短信
- 邮件
- 企业微信 / 钉钉
- 催办、提醒、升级等事件对外送达验证

### 5. 法务级电子签章闭环

- PDF 落章
- 审批材料签章回显
- 验签
- 存证 / 审计留痕

### 6. SLA 运营能力

- SLA 报表
- 升级责任追踪
- 规则可视化与运维说明
- 事件统计与异常排查闭环

## P2：后续增强

### 7. 批量审批增强

- 跨页选择
- 更细的部分失败反馈
- 更复杂的批量策略与权限控制

### 8. 主子流程深链路

- 子流程 + 终止 + 唤醒
- 子流程 + 撤销 + SLA
- 多层主子流程树可视化

### 9. 复杂组合链路人工巡检

- 会签 + 加签 + 驳回 + SLA
- 代理 / 委派 / 离职转办混合链路
- 包容分支 + 子流程 + 动态追加

## 当前判断

- 基础审批平台主链已可用
- 现阶段的主要缺口已从“后端动作能力”转为“前端 E2E、压测、外部通道和法务级闭环”
