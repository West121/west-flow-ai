# 编排自动化基础闭环设计

> 状态：Frozen v1
> 日期：2026-03-22
> 范围：`codex/m0-foundation`

## 1. 目标

本轮把流程平台的“自动化编排”从设计期配置推进到运行期可执行、可追踪、可展示的完整闭环。

本轮交付目标：

- 流程设计器可配置 `超时审批`、`自动提醒`、`定时节点`、`触发节点`
- 后端运行态可自动扫描并执行上述自动化规则
- 审批单详情和流程中心可展示自动动作、提醒记录和执行结果
- 系统管理新增 `通知渠道配置` 与 `触发器管理` 两个独立中文页面

## 2. 冻结范围

### 2.1 本轮必做

- `超时审批`
  - 人工审批节点可设置超时阈值
  - 超时后自动 `APPROVE` 或 `REJECT`
- `自动提醒`
  - 人工审批节点可设置提醒规则、提醒次数、提醒渠道
  - 渠道支持：
    - `IN_APP`：真实发送
    - `EMAIL`：真实发送
    - `WEBHOOK`：真实发送
    - `SMS` / `WECHAT` / `DINGTALK`：适配器接口 + mock provider
- `定时节点`
  - 节点到达后等待指定时间，再自动推进下一步
- `触发节点`
  - 支持 `IMMEDIATE` 和 `SCHEDULED`
  - 执行一次业务触发器逻辑，成功后进入下一步
- `流程中心与审批单详情展示`
  - 自动提醒记录
  - 超时自动审批轨迹
  - 定时节点执行轨迹
  - 触发节点执行轨迹
- `系统管理配置页`
  - `通知渠道配置`
  - `触发器管理`

### 2.2 明确排除项

- 真实短信供应商接入
- 真实企业微信 / 钉钉开放平台签名与租户配置
- 动态构建任务
- AI 驱动触发策略
- 复杂 cron 可视化表达式编辑器
- Flowable 原生 job executor 深度整合

## 3. 设计器与 DSL 设计

### 3.1 审批节点新增配置

审批节点新增两个配置区：

- `timeoutPolicy`
  - `enabled`
  - `durationMinutes`
  - `action`
    - `APPROVE`
    - `REJECT`
- `reminderPolicy`
  - `enabled`
  - `firstReminderAfterMinutes`
  - `repeatIntervalMinutes`
  - `maxTimes`
  - `channels`

说明：

- 超时与提醒都只作用于人工审批节点
- `durationMinutes` 统一先用分钟，不引入复杂表达式
- 渠道数组统一使用枚举值，避免前后端各写一套字符串语义

### 3.2 定时节点设计

新增节点类型：

- `timer`

配置字段：

- `scheduleType`
  - `ABSOLUTE_TIME`
  - `RELATIVE_TO_ARRIVAL`
- `runAt`
- `delayMinutes`
- `comment`

规则：

- `ABSOLUTE_TIME` 直接使用节点配置时间
- `RELATIVE_TO_ARRIVAL` 使用节点进入时间加偏移分钟

### 3.3 触发节点设计

新增节点类型：

- `trigger`

配置字段：

- `triggerMode`
  - `IMMEDIATE`
  - `SCHEDULED`
- `scheduleType`
  - `ABSOLUTE_TIME`
  - `RELATIVE_TO_ARRIVAL`
- `runAt`
- `delayMinutes`
- `triggerKey`
- `retryTimes`
- `retryIntervalMinutes`
- `payloadTemplate`

规则：

- `IMMEDIATE` 到达节点后立即执行
- `SCHEDULED` 到指定时间后执行
- `triggerKey` 由系统管理中的触发器配置页维护

## 4. 后端架构设计

### 4.1 新增包边界

新增两个平台域：

- `orchestrator`
  - 自动作业扫描
  - 作业执行
  - 超时审批
  - 定时推进
  - 触发节点执行
- `notification`
  - 通知渠道配置
  - 通知发送
  - 发送日志
  - mock provider

### 4.2 运行模型

继续复用当前 `processruntime`，不引入第二套任务引擎。

自动化只在当前运行态之上扩展三类记录：

- `调度作业记录`
- `通知发送记录`
- `自动动作执行记录`

执行入口设计：

- `orchestrator` 提供统一扫描接口
- 开发态可通过手动触发接口执行扫描
- 后续接真实定时调度时不改业务执行器接口

### 4.3 超时审批执行规则

- 仅扫描状态为 `PENDING` 的人工任务
- 当前时间超过 `receiveTime + durationMinutes` 视为超时
- 根据节点配置自动调用：
  - `APPROVE`
  - `REJECT`
- 自动动作必须写入：
  - 实例事件
  - 任务轨迹
  - 自动执行记录

### 4.4 自动提醒执行规则

- 仅扫描状态为 `PENDING` 的人工任务
- 超过首次提醒时间后生成提醒
- 每次提醒记录发送结果
- 达到 `maxTimes` 后停止发送
- 渠道结果必须逐条记录成功或失败

### 4.5 定时节点执行规则

- 定时节点到达时先创建调度作业
- 到点后自动推进到下一节点
- 若实例已终止、撤销或节点已失效，则作业标记取消

### 4.6 触发节点执行规则

- 触发节点到达时创建触发作业
- 到点或立即执行时，根据 `triggerKey` 查找触发器定义
- 触发器执行结果：
  - 成功：进入下一节点
  - 失败且仍有重试次数：继续排重试作业
  - 失败且重试耗尽：节点保持失败状态并写事件

## 5. 通知中心设计

### 5.1 渠道枚举

- `IN_APP`
- `EMAIL`
- `WEBHOOK`
- `SMS`
- `WECHAT`
- `DINGTALK`

### 5.2 本轮真实与模拟边界

- `IN_APP`
  - 真实写入平台通知记录
- `EMAIL`
  - 真实发送，使用系统配置中的 SMTP 参数
- `WEBHOOK`
  - 真实发送，按 URL + Header + Body 模板调用
- `SMS`
  - mock provider
- `WECHAT`
  - mock provider
- `DINGTALK`
  - mock provider

### 5.3 通知渠道配置页

系统管理新增独立页面：

- `系统管理 / 通知渠道配置 / 列表`
- `系统管理 / 通知渠道配置 / 详情`
- `系统管理 / 通知渠道配置 / 新建`
- `系统管理 / 通知渠道配置 / 编辑`

字段最小集合：

- 渠道编码
- 渠道类型
- 渠道名称
- 启用状态
- 连接参数
- mock 模式开关

## 6. 触发器管理设计

系统管理新增独立页面：

- `系统管理 / 触发器管理 / 列表`
- `系统管理 / 触发器管理 / 详情`
- `系统管理 / 触发器管理 / 新建`
- `系统管理 / 触发器管理 / 编辑`

字段最小集合：

- `triggerKey`
- `triggerName`
- `triggerType`
  - `WEBHOOK`
  - `JAVA_SCRIPT`
  - `HTTP`
- `enabled`
- `config`
- `remark`

规则：

- 本轮先做最小可执行模型
- `JAVA_SCRIPT` 可挂接 LiteFlow/脚本执行抽象，但先不做复杂脚本编辑器

## 7. 前端展示设计

### 7.1 流程设计器

设计器节点属性面板新增：

- 审批节点：
  - 超时审批
  - 自动提醒
- 定时节点：
  - 定时配置
- 触发节点：
  - 立即触发 / 定时触发
  - 触发器选择
  - 重试配置

### 7.2 流程中心

流程中心列表和详情需要可见：

- 是否存在自动化规则
- 是否发生过自动催办
- 是否因超时被自动审批
- 定时节点执行状态
- 触发节点执行状态

### 7.3 审批单详情

审批单详情补充两个信息区：

- `自动动作轨迹`
  - 自动通过
  - 自动拒绝
  - 定时节点到点执行
  - 触发节点执行成功 / 失败
- `通知发送记录`
  - 渠道
  - 接收人
  - 发送时间
  - 结果
  - 错误信息

## 8. 权限与安全

- 通知渠道配置页：仅流程管理员可见
- 触发器管理页：仅流程管理员可见
- 手动执行编排扫描接口：仅流程管理员可执行
- Webhook / Email 配置中的敏感字段只允许后端保管，前端只回显掩码

- 定时节点
- 触发节点

### Agent B：后端 notification

### Agent C：前端设计器配置

- 审批节点超时 / 提醒配置
- 定时节点 / 触发节点配置
- DSL 与保存 / 详情回填

### Agent D：前端页面与展示

- 通知渠道配置页面
- 触发器管理页面
- 审批单详情自动动作轨迹与通知记录
- 流程中心增强展示

## 10. 验收标准

- 设计器可以配置超时审批、自动提醒、定时节点、触发节点
- 后端可通过统一扫描入口执行自动化规则
- 超时审批会自动推进实例，并留下轨迹
- 自动提醒会生成真实发送记录
- 站内 / 邮件 / Webhook 可真实发送
- 短信 / 微信 / 钉钉通过 mock provider 留下可验证记录
- 系统管理存在独立的通知渠道配置与触发器管理页面
- 审批单详情可查看自动动作轨迹与通知发送记录
